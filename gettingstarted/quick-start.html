Identity Overview #

Steps

  • Find and install the latest version of the nuget package Thermo.Luna.Identity.Authorization.AspNetCore. Add the below code to your configureservices in startup.cs
  • Add the below code to your configureservices in startup.cs
services.AddLunaApiAuthorization(new ApiAuthParameters
            {
                Authority = identityAuthorityUrl,
                ApiName = "DataServerApi", <-- Resource Name
                ApiScopes = new List<string> { "DataServerApi" } <-- Scope Name
            });

  • Authority should point to identity Authority. Can be read from environment variable.

Example: https://identity.luna.cmddev.thermofisher.com

ApiName should be DataServerApi and also the scopes. In case, the service needs to access identity Apis, the ApiScopes can be updated as below




 



services.AddLunaApiAuthorization(new ApiAuthParameters
            {
                Authority = identityAuthorityUrl,
                ApiName = "DataServerApi", <-- Resource Name
                ApiScopes = new List<string> { "DataServerApi", "IdentityServerApi" } <-- Scope Name
            });

  • Add [Authorize] tag on your controller methods to protect the apis. This will validate the incoming token looking for the scope DataServerApi.
 


[Authorize]
public class SequencesController {}

The Identity Team seeded a Scope and API resource both called DataServerApi - this is to be used by Non identity related API’s . Identity api’s will now require the scope of IdentityServerApi.
